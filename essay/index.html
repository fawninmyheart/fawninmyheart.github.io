Git

commit message 规范

commit message包含三部分：Header,Body和Footer

Header是必需的，Body和Footer则可以省略

Header
  1. type（必需）
    用于说明git commit的类别，允许使用下面几个标识。
    feat：新功能
    fix/to：bug修复。to指部分修复，fix指完全修复。
    docs：修改附属文档。
    style：调整了代码书写格式，不影响代码运行。
    refactor：代码重构。
    perf：性能优化
    test：增加测试
    chore：构建过程或辅助工具的变动
    revert：回滚
    merge：代码合并
    sync：同步主线或分支的 Bug，通常用于解决因为合并而引入的问题。
